done

  • done(skip?: boolean): void
    • Defined in database/src/queue.ts:182

    Calling done indicates that the current job is complete (or failed). Generally called from within a 'next' event handler when finished. It will remove the current job from the persistent queue and emit another 'next' event.

    Parameters

    • Default value skip: boolean = false

      Whether to temporarily skip the (probably failed job). If true (default is false) the current job is dropped from the in-memory queue, but will be re-hydrated from persistent storage once the current batch of jobs are complete. Useful to keep a single job from holding up the whole queue, without throwing it out entirely.

    Returns void
